## Codebase Patterns
- Test runner is `uv run --with pytest --with pyright pytest tests/` from the worktree root (no pyproject.toml; no venv on disk — uv resolves a one-off env each call).
- Typecheck is `uv run --with pytest --with pyright pyright`. pyrightconfig.json sets typeCheckingMode=basic, pythonVersion=3.10, includes core/tests/conftest.py.
- Coverage is `uv run --with pytest --with pytest-cov pytest --cov=core.<module>`.
- Frozen dataclasses for immutable value types (Config exception aside — it must be mutable for `replace()`). Use `@dataclass(frozen=True)` and tuple fields, not list.
- One module per concern under `core/`. Tests live at `tests/unit/test_<module>.py`; integration tests at `tests/integration/`.
- Internal helpers prefixed `_` and exported via `from core.<mod> import _helper` for direct unit testing — preferred over heavy monkeypatching when the helper takes parameters that drive its decisions.
- Canonical user-facing strings live as module-level constants (e.g. `WSL_LAUNCH_FIX_MESSAGE`, `NO_CLIPBOARD_WARNING`, `NoBackendError.MESSAGE`) and tests assert on substrings of them to catch silent drift.
- File writes from PromptPal must use LF endings — `newline="\n"` on `open()` and `os.replace` for atomicity.
- conftest.py at repo root prepends the repo root to sys.path so `from core.<mod> import ...` works without installing the package.

  - For `copy_to_clipboard`, monkeypatch `subprocess.run` on the *module's* subprocess attribute (`platform_mod.subprocess.run`) — works because `import subprocess` gives the same module object globally and monkeypatch reverts on test exit.
  - `clip.exe` UTF-8 no-BOM requirement (P1-PLAT-06 / D-9): Python's `str.encode("utf-8")` already omits the BOM — `"utf-8-sig"` would prepend one. Test asserts `not sent.startswith(b"\xef\xbb\xbf")` *and* round-trip decodes correctly.
  - The HOME guard checks the **raw** `os.environ["HOME"]`, NOT the already-resolved `platform.home`. The point of the guard is to catch the regression even if `_resolve_home` already corrected it via passwd — so the user gets the relaunch message.
  - Clipboard priority test must exercise the "all three Unix providers present" case to lock in *order* (not just "xclip works when alone"). Same for "xclip wins over clip.exe on WSL".
  - `pwd` is a Linux/macOS stdlib module — pyright won't flag the import even though it's unavailable on Windows, because the tests run on WSL.
